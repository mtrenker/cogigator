Cogigator Plan Duel Factorio 2.0 / Space Age Pi + Kubernetes

Implementation report

Two blueprints for manufacturing thought.

A visual comparison of the revised Claude Opus 4.8 and GPT-5.5 plans for Cogigator: the Factorio mod where intelligence is bounded by physical machines, datacenter fluids, Pi approval flows, and a GitOps-managed headless server.

Executive read

Claude Opus is the sharper infrastructure conservative: RCON-first, Pi-native, no StatefulSet blast radius, and a bold mechanic where in-game Cognition can govern real LLM spend.

GPT-5.5 is the steadier product engineer: clearer entity families, broader staged transport, stronger testing detail, and a richer path toward multi-fluid datacenter gameplay.

Recommendation: use Claude's infra posture and cognition-as-capability spine, with GPT's entity taxonomy, testing strategy, and staged gameplay maturation.

04 / Shared architecture

The three-layer machine

Both plans converge on a safety-preserving split. The decisive architectural insight is that Pi, bridge, and mod each get one job — and only the mod can touch the world.

Layer 1

Pi extension

  • Read-only LLM tools
  • Status widgets + commands
  • Human approval UI
  • No RCON/game authority

Layer 2

Bridge

  • Narrow HTTP/SSE API
  • RCON adapter + report cache
  • Intent validation + metrics
  • Secrets stay here / Kubernetes

Layer 3

Factorio mod

  • Authoritative world state
  • Zone scans + diagnostics
  • Cognition simulation
  • Apply-time validation

08 / Risk heatmap

Known failure modes

UPS degradation

Tick-spread scans, entity caps, omitted-data markers, and Cognition overload states.

Unsafe mutation

Default read-only, inert intents, bridge + mod validation, in-game ledger, fail closed.

Secret leakage

SealedSecrets only. No tokens in mod, Pi config, sessions, logs, or docs.

Datacenter feels like a tax

Make compute feedback visible, rewarding, tunable, and avoid fluid complexity in MVP.

RCON limits

Derived compact snapshots, chunking, file-output replay, transport adapter abstraction.

Multiplayer authority

Define force ceilings, approval roles, audit entries before Planner mode ships.

09 / Recommended route

Synthesis blueprint

1

Build the safe spine

Use RCON/file fixtures and no persistent cluster mutation. Get the mod scaffold, station/core lifecycle, bounded worksite, and deterministic diagnostics working locally.

2

Make thought physical

Ship a simple capacity model early: scan, attention, memory, planning. Let datacenter machines visibly improve freshness and diagnostic depth before adding complex fluids.

3

Open the Pi cockpit

Add the project-local Pi extension with status, snapshot, and analyze tools. Keep approvals as explicit commands and UI, not autonomous LLM tools.

Best combined MVP

A player builds a Field Station/Core and a minimal datacenter. The station observes a visible rectangle, consumes manufactured capacity, produces deterministic bounded reports, and Pi answers “why is this stalled?” with station/tick/finding citations. No world mutation. No permanent cluster change. The concept proves locality, grounding, physical cognition, and operator safety in one loop.
